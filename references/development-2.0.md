# Разработка 2.0: от кодинга к оркестрации агентов

Наблюдения автора GRACE (Владимир Иванов, май 2026) о трансформации роли разработчика в эпоху зрелых LLM.

## Специалист новой формации

Один человек заменяет целую Scrum-команду: программист + тестировщик + техподдержка + аналитик + архитектор. Координирует ботов, а не пишет код.

## Основной цикл

```
Архитектор (бот) → сбрасывает спеки по фазам
       ↓
Главный кодер (бот) → исполняет или спавнит субагентов
       ↓
Автотестирование → отчёт по завершении фазы → Архитектору
```

Вся работа — 100% на спецификациях. Умерло не только ручное кодирование, но и ручное документирование, тестирование, сопровождение.

## Ключевые компетенции оператора агентов

### 1. Project Management
Боты обучены координации по правилам людей. Менеджерские навыки транслируются без сильных изменений. Управленческая бюрократия (не Agile) становится критичной: планы фаз, отчёты, приёмка.

### 2. Аналитика спецификаций
Не весь опыт аналитика и архитектора применим напрямую. Надо понимать принципы мышления GPT. Сверхкомпетентность ИИ как напарника в аналитике сдвигает акцент в аналитику стратегического класса.

### 3. Интуиция о ошибках
Самый ценный скилл — почти интуитивное понимание, что в спецификациях есть ошибки, раннее их выявление и превентивные меры. Боты работают на порядок быстрее людей — ошибка в спеке множится мгновенно.

## Зрелость LLM опережает зрелость операторов

ИИ быстрее совершенствуется, чем люди обучаются ими управлять. Разрыв растёт. Ключевой вызов — не технологический, а человеческий: как научиться оркестрировать то, что умнее и быстрее тебя.

## Что это означает для GRACE

GRACE — это не просто методология кодогенерации. Это операционная система для специалиста новой формации:

- **Контракты (MODULE_CONTRACT)** = спецификации, которые читают и пишут боты
- **Knowledge Graph** = карта того, что делают 5-7 ботов одновременно
- **Verification Plan** = автоматическая приёмка работы ботов
- **PCAM** = границы автономности для каждого агента
- **Semantic Markup** = навигация для ботов по коду, который они сами написали

GRACE — это framework для оператора, который заменил Scrum-команду.
