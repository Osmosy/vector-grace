# Perplexity как научная метрика GRACE

## Зачем измерять перплексию

Перплексия (perplexity) — фундаментальная ИИ-метрика «удивления» модели относительно контекста. Если модель хорошо понимает контекст, перплексия низкая. Измеряется по вероятностям выбора логитов в GPT.

Автор GRACE использует перплексию для:
- Проверки сходимости GRACE-разметки (обычно на мелких SLM — они жёстче реагируют на нарушение структуры)
- Доказательства, что вэньянь-промптинг читается SLM лучше обычного текста (корреляционные паттерны родные для Attention)
- Поиска «нативного» формата описаний для ИИ

## Self-Aligned Perplexity (arXiv 2502.11779)

Ren et al., 2025 — ключевое улучшение над стандартной перплексией.

### Проблема стандартной перплексии

Стандартная перплексия требует **точного совпадения** с тем, что модель ожидает. Но для одного вопроса может быть много валидных ответов. Модель может быть «знакома» с семантически эквивалентным ответом, но стандартная перплексия этого не видит → **завышает удивление**.

### Self-Aligned Perplexity (SAP)

Измеряет, насколько кандидат-ответ **прилипает к стилю и паттернам рассуждения самой модели**:

```
F(ŷ) = E_{y~M(x)}[s(y, ŷ)]
```

Где:
- `ŷ` — кандидат-ответ (чей-то сгенерированный выход)
- `M(x)` — распределение ответов самой целевой модели
- `s(y, ŷ)` — семантическая близость

**Ключевое отличие**: SAP не штрафует за валидные альтернативные формулировки.
Когда `s = δ` (точное совпадение), SAP = стандартная перплексия.

### Практический вывод для GRACE

- Стандартная перплексия **недооценивает** familiarity модели с хорошим контекстом
- SAP показывает, что важен не точный текст, а **насколько контекст соответствует тому, как модель сама бы его написала**
- Это объясняет, почему встроенная в код разметка (GRACE) эффективнее отдельных MD-файлов: модель «ожидает» увидеть контекст рядом с кодом, а не в отдельном файле

## Surface-Form Brittleness

Даже маленькие переформулировки и перегруппировки текста LLM **значительно деградируют понимание замысла** автора для читателя.

### Механизм

Исходный текст LLM непрерывен, все его части находятся в семантической когерентности. Когда ИИ читает контекст другой LLM, он видит **намного больше семантики, чем в словах** — для него это ключи к векторным комбинациям со скрытым смыслом.

Подтверждено работой «Language models transmit behavioural traits through hidden signals in data»: текст LLM содержит скрытые сигналы, невидимые человеку.

### Что это означает для документации

| Подход | Перплексия | Причина |
|--------|-----------|---------|
| Встроенная разметка (GRACE) | Низкая | Исходный текст LLM сохранён, семантическая когерентность не нарушена |
| Отдельные MD-файлы | Высокая | Текст разрезан, перегруппирован, переформулирован «для людей» |
| RAG/MCP-подбрасывание кусочков | Высокая | Контекст фрагментирован, потеряны скрытые корреляции |
| Поверхностная правка текста LLM | Высокая | Даже «незначительная» по-человечески правка разрушает векторные ключи |

**Главный вывод**: наивно думать, что «незначительная» по человеческим меркам правка текста LLM будет так же незначительной для ИИ. Модель видит в тексте векторные структуры, которые человек не воспринимает.

## Как применять на практике

### Проверка своего контекста

1. Взять мелкую SLM (Qwen3-0.6B, Qwen2.5-0.5B) — они жёстче реагируют на нарушение структуры
2. Подать контекст (разметку, промпт, документацию)
3. Измерить перплексию по логитам
4. Если высокая — контекст не нативный для ИИ, нужно искать другой формат

### Почему GRACE именно такой

- **Knowledge graph (XML)**: нативный формат для Attention — граф = корреляции между векторами
- **Wenyan-prompting**: KEYWORDS вместо предложений — ближе к тому, как Attention обрабатывает токены
- **In-code markup**: сохраняет исходный текст LLM, не нарушает семантическую когерентность
- **GRACE 2.0 GREP-hints**: специальные паттерны для китайских LLM, подтверждённые низкой перплексией

### Когда НЕ использовать перплексию

- Перплексия не заменяет SWE-bench — она измеряет «понимание», а не «решение задач»
- Высокая перплексия иногда полезна: unfamiliar данные могут учить модель новому
- Перплексия на маленькой выборке (50 примеров) — приближение, не точное измерение
