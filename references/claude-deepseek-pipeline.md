# Claude → DeepSeek: Teacher-Student Pipeline

Практический паттерн от автора GRACE (Владимир Иванов, май 2026).

## Проблема

Классическая схема «Claude-архитектор → DeepSeek-кодер» не работает для фронтенда и UI: DeepSeek не может создать нужное качество кода по MD-спецификациям, даже подробным.

## Решение

**Инвертировать поток:** Claude пишет первую версию кода → DeepSeek читает, отлаживает, дорабатывает.

```
Claude (Teacher)        DeepSeek (Student)
      │                       │
      ├─ пишет v1 код ────────→ читает код
      │                       │
      │                       ├─ понимает контекст
      │                       ├─ создаёт векторную репрезентацию (≈дистилляция)
      │                       ├─ отлаживает баги
      │                       └─ дорабатывает
```

## Почему это работает

DeepSeek при чтении кода Claude не просто понимает контекст — он создаёт аналогичную векторную репрезентацию. Это напоминает процесс дистилляции, где Claude = Учитель, DeepSeek = Ученик.

По расходу токенов это может быть даже эффективнее, чем Claude → спеки → DeepSeek, с учётом болтливости Claude при написании спецификаций.

## Критическое условие: семантическая когерентность

Текст от Claude должен быть считан DeepSeek **БЕЗ ИЗМЕНЕНИЙ**.

❌ Не работает:
- Claude пишет код → человек редактирует → DeepSeek читает
- Claude пишет код → сохраняется в MD-файл → DeepSeek читает MD
- Claude пишет код → передаётся через MCP → DeepSeek читает

✅ Работает:
- Claude пишет код с инлайн-разметкой GRACE → DeepSeek читает тот же файл

## Требование к коду

Код должен содержать встроенную документацию (GRACE-разметку):
- `MODULE_CONTRACT` — что делает модуль
- `START_BLOCK` / `END_BLOCK` — навигационные якоря
- `CHANGE_SUMMARY` — история изменений

Никаких внешних MD-файлов, RAG, MCP между Teacher и Student. Текст должен сохранять исходную семантическую когерентность.

## Связь с научным обоснованием

- **Self-Aligned Perplexity** (arXiv 2502.11779): текст должен быть «знаком» целевой LLM
- **Surface-Form Brittleness**: даже малые переформулировки деградируют понимание
- **Hidden Signals**: LLM-текст содержит скрытые сигналы, невидимые человеку

См. `perplexity-metric.md` и `development-2.0.md`.

## Применение в GRACE

```
$grace-init          ← Claude создаёт структуру и пишет v1 код
        ↓
$grace-execute       ← DeepSeek читает код Claude, отлаживает, рефакторит
        ↓
$grace-fix           ← DeepSeek чинит баги по семантической навигации
```

Ключевое: Claude НЕ пишет спецификации для DeepSeek. Claude пишет КОД, который DeepSeek читает напрямую.
