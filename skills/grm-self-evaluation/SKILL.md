# GRM Self-Evaluation

## Суть

DeepSeek V4 и другие модели прошедшие GRPO/PPO на обучении использовали формат:
задача → ответ → GRM-оценка (Principles × Weight × Score).

Вендоры отрезают этот «хвост рефлексии» финальным тюнингом для чата.
Но модель НЕ теряет эту способность — семантика оценок впитана в веса
предыдущих токенов через градиентный спуск.

Если вернуть модели родной формат обучения, качество ответа вырастает.

## Когда использовать

- Сложный анализ и оценка рисков
- Принятие решений где важна калибровка уверенности
- Когда нужно не просто мнение, а самооценка его качества
- Для аудита: видно какие принципы модель считает важными
- Когда стандартный Markdown даёт поверхностный ответ

## Формат промпта

```
[ЗАПРОС]

Ответ оформи СТРОГО как JSON без markdown-обёртки:
{
  "answer": "текстовый ответ",
  "self_evaluation": {
    "principles": [
      {
        "principle": "название принципа",
        "weight": 0.0-1.0,
        "score": 0-10,
        "rationale": "почему такая оценка"
      }
    ]
  },
  "confidence": 0.0-1.0,
  "uncertainties": ["что осталось неясным"]
}
```

## Принципы — как выбирать

Модель сама подберёт principles под задачу. Типичные:
- Clarity / Completeness / Relevance / Conciseness — для анализа
- Correctness / Safety / Efficiency — для кода
- Coverage / Accuracy / Bias-awareness — для исследований

Weight: 0-1, важность принципа относительно других.
Score: 0-10, насколько ответ удовлетворяет принципу.

## Пример — проверено на DeepSeek V4 Pro

Запрос: «Оцени риски внедрения AI-агентов в корпоративную среду»

Ответ:
```json
{
  "answer": "Внедрение AI-агентов сопряжено с рисками утечки данных...",
  "self_evaluation": {
    "principles": [
      {"principle": "Clarity", "weight": 0.3, "score": 9, "rationale": "..."},
      {"principle": "Completeness", "weight": 0.3, "score": 8, "rationale": "..."},
      {"principle": "Relevance", "weight": 0.2, "score": 10, "rationale": "..."},
      {"principle": "Conciseness", "weight": 0.2, "score": 9, "rationale": "..."}
    ]
  },
  "confidence": 0.9,
  "uncertainties": ["Не рассмотрены отраслевые риски"]
}
```

## Почему это работает

1. RL-обучение (GRPO/PPO) требует формальной оценки ответа.
   JSON-structured output + scoring — родной формат для модели.

2. Градиентный спуск идёт «назад»: семантика JSON-оценок впитывается
   в веса токенов рассуждений (CoT), даже если «хвост» потом отрезан.

3. Модель легко переформулирует JSON в Markdown на инференсе,
   но в JSON-формате эффективность ВЫШЕ — ближе к training distribution.

4. Самооценка даёт калибровку: confidence + uncertainties показывают
   где модель «знает что не знает».

## Ограничения

- Не для простых запросов (погода, «как дела»)
- +30-50% токенов на self_evaluation
- Требует парсинга JSON (встроен в Hermes tools)
